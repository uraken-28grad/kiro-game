# Kiro Game - プロジェクト全体像

## 概要

愛媛県の名所をテーマにした **画面切り替え型 2D プラットフォーマーゲーム**。  
React + Pixi.js で構築し、Docker で開発・本番環境を提供します。

## ゲームメカニクス

### プレイヤー操作
- `←` `→` : 左右移動（4px/frame）
- `Space` : ジャンプ（初速 -12、重力 0.6/frame）

### 画面遷移
- 画面右端到達 → 次の画面へ
- 画面左端到達 → 前の画面へ

### 障害物の種類

```
┌──────────────┬─────────────────────────────────────────────┐
│ 種類          │ 挙動                                        │
├──────────────┼─────────────────────────────────────────────┤
│ ハザード(固定) │ ステージ1-4: 固定位置                        │
│ ハザード(移動) │ ステージ5+: 左右に自動移動、壁で反転         │
│ 間欠泉        │ 一定間隔で噴出、噴出中のみ当たり判定         │
│ 雫トラップ    │ 上から周期的に落下                           │
└──────────────┴─────────────────────────────────────────────┘
```

## 座標系

```
  ゲーム画面（4:3 = width x height）

  画面座標系 (描画用)          ステージ座標系 (データ定義用)
  ┌───────────────┐           ┌───────────────┐
  │ (0,0)     → x │           │               │ ↑ y（上が正）
  │ ↓              │           │               │
  │ y              │           │               │
  │                │           │               │
  │          ──────│           │          ──────│ y=0（地面）
  └───────────────┘           └───────────────┘
                                    → x

  変換: screenY = groundY - stageY - height
```

## データ構造

### Screen（1画面分）

```typescript
interface Screen {
  background?: string          // 背景画像パス（4:3全面表示）
  platforms: Rect[]            // 足場の配置
  hazards: Rect[]              // ハザードの配置
  geysers?: GeyserDef[]        // 間欠泉（任意）
  drips?: DripDef[]            // 雫トラップ（任意）
  hazardImage?: string         // この画面固有のハザード画像
  hazardSize?: {w, h}          // この画面固有のハザードサイズ
  goal?: {x, y}               // ゴール座標（最終画面のみ）
}
```

### StageData（ステージ全体）

```typescript
interface StageData {
  id: number
  name: string
  screens: Screen[]
  playerImage?: string         // プレイヤー画像
  playerSize?: {w, h}          // プレイヤーサイズ
  hazardImage?: string         // ハザード画像（ステージ共通）
  hazardSize?: {w, h}          // ハザードサイズ（ステージ共通）
  goalImage?: string           // ゴール画像
  goalSize?: {w, h}            // ゴールサイズ
  geyserImage?: string         // 間欠泉画像
  dripImage?: string           // 雫画像
}
```

## 開発コマンド早見表

| コマンド | 用途 |
|---------|------|
| `docker compose build` | 開発イメージビルド |
| `docker compose run --rm app npm install` | 依存関係インストール |
| `docker compose up -d` | 開発サーバー起動 (localhost:5173) |
| `docker compose run --rm app npm run lint` | Lint |
| `docker compose run --rm app npm run build` | ビルド確認 |
| `docker compose -f docker-compose.prod.yml up -d --build` | 本番ビルド & 起動 (localhost:80) |
